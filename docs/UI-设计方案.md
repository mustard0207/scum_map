## 二、配色方案

### 2.1 当前配色（问题分析）

| 用途 | 当前色值 | 问题 |
|------|---------|------|
| 背景 | `#0f0f23` | 偏紫蓝，缺乏军事感 |
| 主色调 | `#4fc3f7` (亮青) | 太"科技风"，像天气 APP，不像战术工具 |
| 文字 | `#fff` / `#888` | 灰色 `#888` 对比度不足 |
| 卡片 | `rgba(26,26,46,0.9)` | 偏蓝紫，与背景色不协调 |

### 2.2 新配色方案：「野战指挥部」

| 角色 | 色值 | 说明 |
|------|------|------|
| **背景深色** | `#0C1018` | 纯正的深夜黑，带一丝军绿底调 |
| **面板/卡片** | `#141C26` | 比背景亮一档，深靛蓝灰 |
| **边框线** | `#2A3544` | 低存在感的结构线 |
| **主强调色** | `#C8A94E` | **军用黄铜金** — 核心操作、选中态、坐标高亮 |
| **辅助强调色** | `#6B8F3A` | **军绿** — 安全/完成状态、地图标记 |
| **警示色** | `#C75050` | **暗红** — 删除、警告 |
| **主文字** | `#E8E4DB` | 微暖的象牙白，比纯白柔和，减少视觉疲劳 |
| **次文字** | `#7A8599` | 冷灰蓝，信息密度区域的辅助文字 |
| **禁用文字** | `#4A5568` | 不可操作元素 |

## 三、字体方案

### 3.1 微信小程序字体限制说明

微信小程序**不支持**自定义 @font-face 加载外部字体（Google Fonts 等），只能使用系统内置字体或通过 `wx.loadFontFace` 加载**托管在自己域名上**的字体文件。考虑到我们零成本无服务器的约束，采用**纯系统字体方案**。

### 3.2 字体栈

| 用途 | 字体栈 | 说明 |
|------|--------|------|
| **坐标/数据** | `'Menlo', 'Courier New', 'Consolas', monospace` | 等宽字体，数字对齐，军事数据感 |
| **中文正文** | `-apple-system, 'PingFang SC', 'Helvetica Neue', sans-serif` | 系统默认中文，清晰锐利 |
| **英文标题** | `'DIN Alternate', 'DIN', 'Helvetica Neue', sans-serif` | DIN 系列自带工业/军事气质（系统预装） |

### 3.3 字号规范

| 层级 | 大小 | 用途 |
|------|------|------|
| 标题 | 34rpx (17px) | 导航栏标题 |
| 正文 | 28rpx (14px) | 卡片标题、按钮文字 |
| 辅助 | 24rpx (12px) | 描述文字、标签 |
| 数据 | 22rpx (11px) | 坐标数字、缩放级别、时间戳 |
| 微型 | 20rpx (10px) | 角标、版本号 |

- 用黄铜金色替代原来的红色，更符合战术风格

## 五、交互规范

### 5.1 触摸反馈

| 元素 | 默认态 | 触摸态 | 过渡时间 |
|------|--------|--------|---------|
| 卡片 | border: #2A3544 | border: #C8A94E + 轻微上移 | 200ms |
| 按钮 | bg: #C8A94E | bg: #B89A3E (暗 10%) | 150ms |
| 底栏图标 | color: #7A8599 | color: #C8A94E | 150ms |
| 列表项 | bg: transparent | bg: rgba(200,169,78,0.05) | 150ms |

### 5.2 微交互

| 场景 | 效果 |
|------|------|
| 选点成功 | 短振动 (`wx.vibrateShort`) + 标记弹窗淡入 (200ms) |
| 缩放切换 | 瓦片瞬间切换（无动画，保持性能） |
| 页面进入 | 导航栏从上滑入 + 内容区从下淡入 (250ms) |
| 按钮点击 | 即时反馈，无延迟 |

## 六、emoji 图标替换方案

> **UI/UX Pro Max 核心规则：不要用 emoji 做 UI 图标**

当前使用的 emoji 图标需要替换为**文字图标**或**Unicode 符号**（微信小程序不支持 SVG inline，需要使用 icon font 或图片资源）。

### 方案 A：纯文字 + Unicode 符号（零资源依赖）

| 原 emoji | 替换方案 | 说明 |
|----------|---------|------|
| 🗺️ (地图) | `MAP` 文字标签 | 首页卡片用文字标题 |
| 📍 (选点) | `◆` 或 `⊕` | Unicode 几何符号 |
| 📤 (分享) `⎋` | 或 `↗` | 箭头符号 |
| 🎯 (重置) | `⊙` 或 `⌂` | 定位符号 |
| 🗺️ (logo) | `▲` 三角形 | 军事/战术感 |
| ‹ (返回) | `‹` | 保持不变，已够好 |

### 方案 B：Iconfont 图标库（推荐，效果更好）

使用阿里巴巴 iconfont，打包成微信小程序的自定义字体：
1. 在 iconfont.cn 选择军事/地图/工具类图标
2. 下载为 Base64 编码的字体文件
3. 在 app.wxss 中通过 `@font-face` 引入

**推荐图标风格**：线性 (Outlined)、1.5px 线宽、圆角端点

## 九、方案对比预览

### 当前方案
- 色调：紫蓝 + 亮青 → 科技感，但偏"天气 APP"
- 图标：emoji → 低端感
- 文字：纯白 + 灰 #888 → 对比度不统一

### 新方案「野战指挥部」
- 色调：深靛 + 黄铜金 + 军绿 → 战术/军事硬核感
- 图标：Unicode 符号或 iconfont → 简洁专业
- 文字：象牙白 + 冷灰蓝 → 层次分明，长时间使用不疲劳
