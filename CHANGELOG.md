# 更新日志

## v7.7 (2026-06-26)

### 优化 (WXS 高清与高性能重构)
- **解决地图模糊问题**：彻底移除瓦片地图容器上的 `transform: scale` 缩放属性，改为在 WXS 中直接通过底层渲染节点更新物理尺寸，确保地图缩放时的高清渲染（零锯齿/原画质）。
- **解决滑动卡顿问题**：引入 WXS 渲染接管机制（`gestures.wxs`），将所有手势拖拽、惯性计算、捏合缩放从 JS 业务层完全迁移至视图层（渲染线程）独立运算，告别因高频 `setData` 通信造成的卡顿，实现满帧极致顺滑。
- **动态坐标补偿机制**：重构 `onMarkerTap` 的屏幕像素反推算法，在零通信成本的基础上兼顾了业务层标记窗口原本对点击坐标 `screenX/Y` 的依赖。
- **渲染中断保护**：在 WXS 与 JS 通信机制中增设了 `abort` 强制中断信号，完美防止因惯性滑动与 JS API 坐标跳转（如“回到中心”、“定位至标记”）同时发生导致的画面抖动和坐标穿透回弹。

## v7.6 (2026-06-26 01:49:06)

### 新增
- **Z3+Z4+Z6 瓦片分层加载**：根据缩放级别动态切换瓦片层级
  - Z2 (scale < 1.0)：本地 JPG 底图
  - Z3 (1.0 ~ 1.5)：jsDelivr 网络 webp，2×2 网格
  - Z4 (1.5 ~ 3.0)：jsDelivr 网络 webp，4×4 网格
  - Z6 (scale ≥ 3.0)：jsDelivr 网络 webp，16×16 网格，Z4 作为兜底
- **缩放拖拉条**：右侧垂直拖拉条，以十字准星为中心缩放，支持鼠标和手指操作
- **本地缓存机制**：网络瓦片首次加载后自动缓存到用户本地，后续直接读取

### 优化
- 修复 `_clampOffset` 时序 bug（scale 先更新再钳制边界）
- 坐标面板与全局视角按钮位置互换
- 坐标标签从"经度/纬度"改为"X/Y"，与游戏坐标格式一致
- 地图区域使用动态 top/bottom 定位，消除底栏黑边
- 移除关于页面和 scum-map.com 数据来源引用
- 添加 debug 日志和 `wx.getWindowInfo` 防御
