# 更新日志

## v1.0.0 (2026-06-29)

### 新增
- **自定义标记筛选**：地点筛选面板新增"自定义标记"大类
  - 支持按类型筛选：📌 无类型 / 🏠 房屋 / 🚗 载具 / 📦 储物箱
  - 大类总开关：一键开启/关闭所有自定义标记
  - 小类独立开关：可按类型显示/隐藏，各类型显示对应标记数量
  - 大类开关联动：所有小类关闭时大类自动关闭，开启大类时小类全开
  - 筛选不影响数据：被隐藏的标记保留在存储中，取消筛选后恢复显示
- **标记有效期系统**：自定义标记支持维护时效管理
  - 每个标记记录放置时间（精确到秒）
  - 信息窗口显示剩余时间（X天X小时X分）
  - 快过期的标记在地图上红白闪烁边框提醒（默认剩余 <2 天）
  - "刷新有效期"按钮一键重置放置时间
  - 可配置各类型默认过期天数（房屋 15 天、载具 10 天、箱子 15 天、无类型永不过期）
  - 可配置最小提醒天数阈值
- **清空所有标记**：管理菜单新增"清空所有标记"功能，带二次确认弹窗
- **标记上限提升**：自定义标记上限从 50 提升到 100，与 POI 标记独立计数

### 修复
- **分享遗漏标记**：修复底栏分享时因 `selectedMarker` 残留导致只分享单个标记的 bug（增加 `showInfoWindow` 检查）
- **弹窗误关闭**：修复信息窗口、导入弹窗、备份抽屉、保存确认弹窗点击内容区域意外关闭的 bug（`catchtap=""` → `catchtap="noop"`）

### 优化
- **术语优化**：筛选面板"POI 筛选"改为"地点筛选"，分享卡片"分类POI"改为"探索点"
- **菜单重命名**：底部栏"备份"改为"管理"，"导入标记"改为"导入自定义标记"，"导出标记"改为"导出自定义标记"
- **首页描述**：改为"查看和标记游戏地图位置"
- **版本号重置**：正式发布版本 v1.0.0
- **技术文档**：新增 `catchtap=""` 避坑备忘案例（8.4 节）

## v7.9 (2026-06-26)

  - 手动清除筛选或重置视图时同步更新存储

  - "覆盖前备份"导出本地旧存档到剪贴板
  - 用户在分享页自行操作标记/筛选时自动接管为本地数据
- **POI 标记分享**：POI 信息窗口新增分享按钮（无删除按钮），可单独分享任意 POI 点位
- **分享增强**：
  - 信息窗分享按钮 → 分享单个标记（含类型信息）
  - 底部栏分享按钮 → 分享自定义标记 + POI 筛选分类 ID
  - 接收方自动恢复 POI 筛选状态并加载对应分类点位
- **坐标输入提示**：优化提示文案为"将游戏内地图的坐标粘贴到此处，即可在离线地图上标记该位置，方便路线规划与点位记录"

### 合规整改
- **首页"即将推出"占位按钮**：删除未完工的"更多工具"卡片，避免测试版驳回
- **分享标题静态化**：单标记分享标题改为固定文案"我在SCUM地图上标记了一个位置"，阻断 UGC 敏感词传播
- **坐标提示文案去敏感词**：去掉"Ctrl+C"、"游戏内地图按"等可能被误判为外挂辅助的措辞

- **统一命名**：所有界面"SCUM 地图"统一为"SCUM游戏地图"
- **用户隐私保护指引**：已在微信公众平台提交剪贴板隐私声明

- **渲染限流**：全局视角每区每小类最多 10 个点，放大 ≥6x 显示全部，硬上限 1000
- **清除筛选按钮**：一键清除所有已选分类

### 修复
- **筛选面板误关闭**：重构事件结构，遮罩层与面板分离，消除嵌套 `wx:for` 下 `catchtap` 不可靠的问题
- **筛选开关无视觉反馈**：改用 `activePoiMap` 对象查找表替代 `indexOf`，解决 WXML 模板中数组方法不可靠的问题

## v7.7 (2026-06-26)

### 修复
- **坐标输入弹窗误关闭**：修复点击输入框或对话框区域时弹窗意外关闭的问题。根因是 `catchtap=""`（空字符串）在微信小程序中无法可靠拦截事件冒泡，改为 `catchtap="noop"`（真实方法引用）确保 tap 事件不冒泡到 mask 层。

### 优化 (WXS 高清与高性能重构)
- **解决地图模糊问题**：彻底移除瓦片地图容器上的 `transform: scale` 缩放属性，改为在 WXS 中直接通过底层渲染节点更新物理尺寸，确保地图缩放时的高清渲染（零锯齿/原画质）。
- **解决滑动卡顿问题**：引入 WXS 渲染接管机制（`gestures.wxs`），将所有手势拖拽、惯性计算、捏合缩放从 JS 业务层完全迁移至视图层（渲染线程）独立运算，告别因高频 `setData` 通信造成的卡顿，实现满帧极致顺滑。
- **动态坐标补偿机制**：重构 `onMarkerTap` 的屏幕像素反推算法，在零通信成本的基础上兼顾了业务层标记窗口原本对点击坐标 `screenX/Y` 的依赖。
- **渲染中断保护**：在 WXS 与 JS 通信机制中增设了 `abort` 强制中断信号，完美防止因惯性滑动与 JS API 坐标跳转（如“回到中心”、“定位至标记”）同时发生导致的画面抖动和坐标穿透回弹。

- **本地缓存机制**：网络瓦片首次加载后自动缓存到用户本地，后续直接读取
