# 更新日志

## v7.8 (2026-06-26)

### 新增
- **POI 分类筛选功能**：支持按分类显示/隐藏系统 POI 标记点
  - 精选模式：地堡、载具、辐射 3 个可展开大类 + 警局、药店、加油站 3 个直选小类
  - 完整模式：12 个大类全部可展开，102 个小类可自由筛选
  - 最多同时开启 5 个小类
  - 默认显示前哨站，可手动关闭或清除
- **POI 数据来源升级**：从 scum-map.com GraphQL API 获取完整分类数据
  - `category-map.js`：102 个分类定义（section/emoji/cnName）
  - `分类/*.js`：102 个点位文件，7614 个 POI 标记点
  - 按 section 合并为 12 个数据文件，按需加载
- **标记 emoji 化**：所有 POI 标记使用 emoji 图标，不同分类用不同 emoji 区分
- **标记渲染优化**：用户标记使用 📌（图钉），POI 标记使用各自 emoji
- **渲染限流**：全局视角每区每小类最多 10 个点，放大 ≥6x 显示全部，硬上限 1000
- **清除筛选按钮**：一键清除所有已选分类
- **数据更新脚本**：`tools/merge-poi-data.js` 将 102 个分类文件合并为 12 个 section 文件

### 修复
- **筛选面板误关闭**：重构事件结构，遮罩层与面板分离，消除嵌套 `wx:for` 下 `catchtap` 不可靠的问题
- **筛选开关无视觉反馈**：改用 `activePoiMap` 对象查找表替代 `indexOf`，解决 WXML 模板中数组方法不可靠的问题

## v7.7 (2026-06-26)

### 修复
- **坐标输入弹窗误关闭**：修复点击输入框或对话框区域时弹窗意外关闭的问题。根因是 `catchtap=""`（空字符串）在微信小程序中无法可靠拦截事件冒泡，改为 `catchtap="noop"`（真实方法引用）确保 tap 事件不冒泡到 mask 层。

### 优化 (WXS 高清与高性能重构)
- **解决地图模糊问题**：彻底移除瓦片地图容器上的 `transform: scale` 缩放属性，改为在 WXS 中直接通过底层渲染节点更新物理尺寸，确保地图缩放时的高清渲染（零锯齿/原画质）。
- **解决滑动卡顿问题**：引入 WXS 渲染接管机制（`gestures.wxs`），将所有手势拖拽、惯性计算、捏合缩放从 JS 业务层完全迁移至视图层（渲染线程）独立运算，告别因高频 `setData` 通信造成的卡顿，实现满帧极致顺滑。
- **动态坐标补偿机制**：重构 `onMarkerTap` 的屏幕像素反推算法，在零通信成本的基础上兼顾了业务层标记窗口原本对点击坐标 `screenX/Y` 的依赖。
- **渲染中断保护**：在 WXS 与 JS 通信机制中增设了 `abort` 强制中断信号，完美防止因惯性滑动与 JS API 坐标跳转（如“回到中心”、“定位至标记”）同时发生导致的画面抖动和坐标穿透回弹。

## v7.6 (2026-06-26 01:49:06)

### 新增
- **Z3+Z4+Z6 瓦片分层加载**：根据缩放级别动态切换瓦片层级
  - Z2 (scale < 1.0)：本地 JPG 底图
  - Z3 (1.0 ~ 1.5)：jsDelivr 网络 webp，2×2 网格
  - Z4 (1.5 ~ 3.0)：jsDelivr 网络 webp，4×4 网格
  - Z6 (scale ≥ 3.0)：jsDelivr 网络 webp，16×16 网格，Z4 作为兜底
- **缩放拖拉条**：右侧垂直拖拉条，以十字准星为中心缩放，支持鼠标和手指操作
- **本地缓存机制**：网络瓦片首次加载后自动缓存到用户本地，后续直接读取

### 优化
- 修复 `_clampOffset` 时序 bug（scale 先更新再钳制边界）
- 坐标面板与全局视角按钮位置互换
- 坐标标签从"经度/纬度"改为"X/Y"，与游戏坐标格式一致
- 地图区域使用动态 top/bottom 定位，消除底栏黑边
- 移除关于页面和 scum-map.com 数据来源引用
- 添加 debug 日志和 `wx.getWindowInfo` 防御
